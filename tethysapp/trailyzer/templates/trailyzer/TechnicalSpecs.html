Software Design Specifications

This section includes information regarding the design of the software, including purpose, code design, required data files and structure, and a workflow process. The software is designed to be used as a web application, hosted in the Tethys environment. Once installed in the Tethys environment the software will perform with minimal maintenance, although user customizations could be made in the code. Instructions to setup a Tethys environment on a server, as well as to install the app from source can be found on the Tethys website at tethysplatform.org.

Purpose

This software was designed to elevation profiles for any location in the world. In conjunction with this application geoprocessing functions were created and hosted on an ArcGIS server, where the profiles are created based on user input. Geoprocessing functionality was created in ArcMAP and is hosted on an ArcGIS server as a geoprocessing service. The model that was created and hosted uses the Stack Profile Tool, and can be seen in the following image. The Stack Profile Tool was linked to ESRI's World Map service using a BYU Organizational account to allow for access to the World DEM for processing. The required input to link to the service is embedded within the tool, so that no user verification is necessary.

Code Structures

The code structure is patterned after the default Tethys stack, based on the Model-View-Controller pattern of programming, with the default Tethys stack being used to originally create the software. A separate file exists for the base HTML page, the main HTML page, and the JavaScript used by the map. All required data for display, including the .kml files, are included in the default application programming structure developed for the Tethys Platform.

Main Data Files

It is necessary to install the Tethys application from source in a Tethys environment in order to run. No data is required by the end user for the proper use of the app. The user will be required to input a line of interest, which will be passed to the geoprocesser for analysis.

Key Functions

For assistance in reviewing the source code there are several primary components to the key functionality of the App that will be reviewed here. Each reviewed piece of functionality is essential to understand how to update or incorporate changes into this app in the future. In the app.py file the URL map function is defined to the map.

There are several .html files included in the app, base.html and home.html, and documentation .html pages. Each is important in its functionality. The file base.html populates the app navigation items, as well as defining defaults such as the app banner and image, as well as providing links to JavaScript files that are hosted online and provide functionality. The home.html file defines the location of the app content, including the map location. The home.html file references the content of the JavaScript file by calling the map id inside a division tag.

The main.js file provides unique functionality for this application. The main.js file references several other JavaScript files, which are hosted online by ESRI, and are linked through the base.html file, and listed prior to the main.js file. The main.js file provides functionality for drawing a line on the map, and then passing the line to the geoprocessor, where it is analyzed and returns profile elevation information.

User Interface Elements

The app provides a user interface that consists of the map and two buttons along the top to determine if the user is going to input a polyline or a freehand polyline. The sidebar can be collapsed to increase the width of the map. The user interface can be seen in the following picture. After a polyline has been drawn the elevation profile will be displayed underneath the map area.
